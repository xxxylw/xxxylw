# xxxylw GitHub Profile Design

## Goal

Create a GitHub profile README that feels personal, warm, and technical: a cozy pixel-farm coding workshop where robotics, graphics, and deep learning projects can live without feeling like a generic resume page.

## References

- `VoltAgent/awesome-design-md` reference direction: use the Dell 1996 reference for retro web artifacts, hard-edged GIF energy, and playful homepage structure.
- Use modern developer-tool restraint from Cursor-style references: calm writing, clear hierarchy, and minimal clutter.
- Do not copy any copyrighted game characters, logos, or exact brand art. The pixel farm banner uses original characters and an original helper robot.

## Visual Direction

- Mood: cozy pixel RPG, farm workshop, small coding lab, handmade and friendly.
- Palette: grass green, wheat yellow, sky blue, earth brown, sunset orange, and small teal terminal accents.
- Texture: pixel-art GIF/PNG assets, simple badges, light retro web details.
- Shape: square pixel assets, no glossy modern card-heavy UI.
- Motion: short looping GIFs, slow enough to read, small enough for GitHub README performance.

## README Structure

1. Welcome GIF hero.
2. Bilingual name line.
3. Short English introduction.
4. Current status as a graduate student at Guangzhou University.
5. Focus areas and learning trail.
6. Project shelf for future highlighted work.
7. Contact link.

## Asset Inventory

- `assets/github-profile/welcome.gif`: main welcome animation.
- `assets/github-profile/welcome-01.png` through `welcome-04.png`: source frames.
- `assets/github-profile/welcome-palette.png`: generated palette for the current GIF build.

## Content Voice

- English-first, concise, friendly.
- Include Chinese where it helps identity and context.
- Avoid overclaiming. Use "learning", "exploring", and "building" for areas still in progress.
- Keep project descriptions grounded in actual repositories once they are available.
